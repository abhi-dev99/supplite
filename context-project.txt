Project Context Log - Supplite
Date: 2026-04-10

Technical Change Log
- Implemented national-scope real estate heatmap pipeline using Census ACS endpoints with ZCTA geography in backend heatmap services.
- Added ACS year auto-resolution logic to gracefully fallback when requested latest year (for example 2025) is not fully published; current resolved pair is 2024 vs 2023.
- Added full export workflow for training data generation at national scale via backend/scripts/export_real_estate_training_data.py.
- Fixed Gazetteer centroid ingestion for nationwide mapping by using modern Census path format and delimiter-aware parsing.
- Added model training/scoring script backend/scripts/train_real_estate_model.py to generate scored_real_estate_demand_full.csv from full training export.
- Updated scored heatmap API behavior to prefer full scored CSV output when available.
- Updated frontend heatmap fetch strategy and limits to support national/full-US rendering; map layers kept toggleable (Heatmap, Stores, DC Radius) for clarity.
- Updated backend README with full pipeline commands and operational notes.

English Summary
- We upgraded the map from a small sample to a large, US-wide data pipeline.
- The backend now tries to use the newest years you ask for, and if Census has not published them yet, it automatically uses the closest available years.
- We added scripts so you can export the full dataset, train a model, and generate a scored file used directly by the map.
- The map can now show broader US coverage while still letting you hide noisy layers.
- Documentation was updated so setup and reruns are straightforward.

Safety / Sensitive Data Notes
- No new environment files were added.
- CSV/parquet/data exports are ignored by .gitignore and are not intended to be committed.
- Virtual environment and cache artifacts are excluded from commit scope.
