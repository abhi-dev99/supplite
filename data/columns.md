## File: `sku_catalog.csv`
**Purpose:** Static SKU metadata — one row per product.

| Column | Type | Description |
|--------|------|-------------|
| `sku_id` | string | Unique product identifier. Format: `{BRAND_PREFIX}-{CATEGORY_SHORT}-{NUM}` (e.g., `PB-BLANKET-42`) |
| `product_name` | string | Human-readable product name with finish/color (e.g., "Throw Blanket, Cognac") |
| `brand` | string | WSI brand: Pottery Barn, West Elm, Williams Sonoma, Pottery Barn Kids, Rejuvenation |
| `category` | string | Product category: Bedding, Furniture, Decor, Lighting, Kitchen, Entertaining, Outdoor, Accessories |
| `price` | float | Retail unit price in USD |
| `cost_price` | float | COGS per unit in USD |
| `lead_time_days` | int | Vendor lead time in days (28–84). Time from PO placement to DC receipt |
| `home_metro` | string | Primary metro market for this SKU. Scenario effects are strongest here |
| `scenario_type` | char | Scenario code A–K or R. See [Scenario Taxonomy](#scenario-taxonomy-a-k--r) |
| `scenario_label` | string | Human-readable scenario name (e.g., "viral_spike") |

## File: `sku_daily_signals.csv`

**Approximate size:** 120,450 rows (30 SKUs × 11 metros × 365 days)

| Column | Type | Description |
|--------|------|-------------|
| `day_index` | int | Sequential day number (0–364) |

| `search_index` | float | Google Trends search index (0–100). Leads sales by 7–14 days |
| `housing_permits` | int | Monthly new housing permits for this metro (repeated daily within each month). Source: US Census Building Permits Survey |
| `median_income` | int | Census B19013 median household income for this metro (constant per metro) |
| `holiday_flag` | int | 1 if the current date is within a holiday search ramp window, 0 otherwise |
| `holiday_name` | string | Name of the active holiday (e.g., "Christmas", "Thanksgiving") or empty string |
| `scenario_type` | char | Scenario code (A–K or R) for traceability |
| `sales_velocity_7d` | float | 7-day sales velocity: % change in 7-day rolling average vs. prior 7 days. Positive = accelerating demand |
| `sales_velocity_30d` | float | 30-day sales velocity: % change in 30-day rolling average vs. prior 30 days |
| `search_velocity_7d` | float | 7-day search velocity: % change in search index vs. 7 days ago |
| `permit_velocity_30d` | float | 30-day permit velocity: % change in 30-day rolling permits vs. prior 30 days |
| `composite_score` | float | **Weighted demand risk score (0–100)**. See [Composite Scoring Model](#composite-scoring-model) |
| `income_re_score` | float | Sub-score: Income + Real Estate combined (0–100). 25% of composite |
| `holiday_search_score` | float | Sub-score: Holiday search prediction (0–100). 20% of composite |
| `base_search_score` | float | Sub-score: Non-holiday Google Trends (0–100). 15% of composite |
| `sales_velocity_score` | float | Sub-score: Past sales velocity (0–100). 40% of composite |

## Composite Scoring Model

The composite demand score (0–100) weights four signal families:

| Component | Weight | Description | Score Range |
|-----------|--------|-------------|-------------|
| **Sales Velocity** | 40% | 7-day WoW change in rolling daily sales average. Positive acceleration = higher score. This is the strongest signal because trend ≠ sales — actual purchase data must anchor the forecast | 0–100 |
| **Income + Real Estate** | 25% | Combined sub-score: (a) Census B19013 median income normalized against the $120K premium threshold (50 pts max), (b) 30-day housing permit velocity (50 pts max), (c) +10 correlation bonus when income > $120K AND permits are rising. Only high-income real estate drives WSI premium demand | 0–100 |
| **Holiday Search Prediction** | 20% | Pre-holiday search ramp detection. During the ramp-up window (7–35 days before a holiday), this score rises linearly toward the peak multiplier. Critical for Thanksgiving/Christmas planning in Kitchen, Entertaining, and Decor categories | 0–100 |
| **Search Trends (non-holiday)** | 15% | Baseline Google Trends virality. Capped at 15% because search does not directly equal sales — viral buzz can create overstock risk if weighted too heavily | 0–100 |

**Formula:**
```
composite = 0.40 * sales_velocity_score
          + 0.25 * income_re_score
          + 0.20 * holiday_search_score
          + 0.15 * base_search_score
```

### Income + Real Estate Correlation Logic

The income+RE sub-score enforces a clear correlation:

1. **Income component** (max 50 pts): `min(1.0, median_income / $120,000) * 50`
   - San Francisco ($136.7K) → 50 pts
   - Miami ($59.6K) → 24.8 pts

2. **Real Estate component** (max 50 pts): Normalized 30-day permit velocity
   - Rising permits → higher score
   - Flat/declining → lower score

3. **Correlation bonus** (+10 pts): Only awarded when BOTH income ≥ $120K AND permits are rising >5%
   - This ensures that only high-income metros with active new construction are flagged as premium demand territory
   - Rationale: A new $800K home in San Francisco is far more likely to drive a Williams Sonoma purchase than a $200K starter in Houston

## Scenario Taxonomy (A–K + R)

Each scenario represents a distinct demand pattern that the system must detect and respond to. The 30 SKUs collectively cover all 11 named scenarios plus routine items.

### A — Viral Spike
**Trigger:** Social media/influencer event causes demand to 2–4x in days.
**Signal pattern:** Search index leads by 7–14 days, then sales explode. Search velocity +200–800%.
**Risk:** STOCKOUT_RISK in home metro.
**Action:** Expedite emergency order. Example SKU: `PB-BLANKET-42`.
**Test validates:** System detects search-to-sales lead time and triggers stockout alert before inventory runs out.

### B — Silent Overstock
**Trigger:** A previously trending product's demand has peaked and is now silently declining.
**Signal pattern:** Search declining 15–25% over 8 weeks. Sales following with a lag. Inventory coverage rising above 3x lead time.
**Risk:** OVERSTOCK_RISK.
**Action:** Pause replenishment, evaluate 15% markdown.
**Test validates:** System catches declining demand velocity and flags excess inventory before it becomes a markdown event.

### C — Housing Leading Indicator
**Trigger:** Housing permits surge +30–40% in a metro area, predicting furniture demand 60–90 days later.
**Signal pattern:** Permit velocity spikes first, search follows weeks later, sales arrive last.
**Risk:** WATCH → reclassify to action as demand materializes.
**Action:** Pre-position inventory to the relevant DC. Example: Phoenix permits up → ship to Litchfield Park DC.
**Test validates:** The 60-day lead-lag relationship between permits and sales, and the income correlation (high-income RE matters more).

### D — Steady Baseline (Control)
**Trigger:** None — this is the "nothing is happening" control case.
**Signal pattern:** All signals within ±10% of rolling average. Flat composite score.
**Risk:** OK.
**Action:** Maintain current replenishment cadence.
**Test validates:** System does NOT generate false positives. Low alert fatigue.

### E — Seasonal Surge
**Trigger:** Holiday-driven demand spike (Thanksgiving/Christmas for Kitchen/Entertaining, Memorial Day for Outdoor).
**Signal pattern:** Holiday search score rises 3–5 weeks before the holiday. Season multiplier amplified 1.5–2x. Sales follow.
**Risk:** WATCH during ramp-up; OK off-season.
**Action:** Pre-stock for anticipated holiday demand. Coordinate with DC for staging.
**Test validates:** Holiday prediction window detects demand 3–5 weeks early, allowing lead-time-compatible action.

### F — Multi-Signal Convergence
**Trigger:** All three signal families (search + permits + income) align upward simultaneously.
**Signal pattern:** Composite score rises steeply because all four sub-scores are elevating. Strongest in high-income metros.
**Risk:** WATCH (with confidence).
**Action:** Increase order quantity 20%. High confidence because signals from independent sources agree.
**Test validates:** System correctly identifies multi-signal agreement and assigns higher confidence than single-signal cases.

### G — Post-Peak Overstock
**Trigger:** A product that surged during the holiday season is now overstocked as demand normalizes.
**Signal pattern:** Holiday search score drops to 0. Sales velocity turns negative. Days of supply exceeds 3x lead time.
**Risk:** OVERSTOCK_RISK.
**Action:** Cancel pending orders. Apply 15% markdown. Reallocate shelf space.
**Test validates:** Post-holiday cliff detection and the transition from SURGING → FADING surge flag.

### H — Sudden Collapse
**Trigger:** Demand drops precipitously (e.g., product recall, bad press, category disruption).
**Signal pattern:** 7-day velocity drops 50–85% in under 2 weeks. Surge flag flips to FADING.
**Risk:** WATCH → escalate to action.
**Action:** Halt all replenishment immediately. Investigate root cause.
**Test validates:** Rapid collapse detection speed — system should flag within 7 days of onset.

### I — Slow Burn
**Trigger:** Gradual, sustained demand growth over 6+ months.
**Signal pattern:** Sales velocity consistently +3–5% MoM. Search trending upward slowly. No single spike event.
**Risk:** OK → WATCH (if inventory coverage drops below 1.3x lead time).
**Action:** Increase replenishment cadence by 10–15%. Monitor for acceleration.
**Test validates:** System distinguishes slow organic growth from noise and recommends proactive restocking.

### J — Flash in the Pan
**Trigger:** Demand spikes sharply for ~1 week, then returns to baseline.
**Signal pattern:** 3–5 day spike in search and sales, then immediate normalization. Surge flag goes SURGING → STEADY within 2 weeks.
**Risk:** OK (after initial WATCH).
**Action:** Do NOT overreact. Monitor for recurrence, but do not expedite orders.
**Test validates:** System correctly de-escalates a transient spike. Prevents overstock from panic ordering.

### K — Holiday Gifting
**Trigger:** A SKU that sells primarily during gifting holidays (Christmas, Mother's Day, Father's Day).
**Signal pattern:** Very low baseline (0.6x normal). Massive holiday multiplier (2.5x). Holiday search score drives 80%+ of composite.
**Risk:** Seasonal — WATCH during ramp, OK otherwise.
**Action:** Build inventory 6 weeks before holidays, drawdown aggressively after.
**Test validates:** Extreme seasonality detection. The scoring model correctly attributes demand to the holiday search component.

### R — Routine
**Trigger:** Standard demand patterns with seasonal variation but no extreme scenarios.
**Signal pattern:** Normal noise, seasonal curves, and metro-level income modulation, but no special events.
**Risk:** Primarily OK, occasionally WATCH from natural velocity shifts.
**Action:** Standard automated replenishment.
**Test validates:** Baseline model behavior. Ensures the system handles the ~60% of SKUs that are "normal" without generating noise.
